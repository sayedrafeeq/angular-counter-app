📊 Angular Counter App - Learning Notes

Project Overview

This is a modern Angular 18 counter application that demonstrates current best practices in Angular development. It showcases modern architectural patterns and reactive programming techniques.

Key Learning Points

  1. Standalone Components - No NgModules required. Cleaner, more modular architecture
  2. Angular Signals - New reactive API for state management instead of RxJS BehaviorSubject
  3. Modern Bootstrap - Uses bootstrapApplication() instead of platformBrowserDynamicModule()
  4. TypeScript 5.5 - Latest version with improved type safety
  5. Inline Styling - Component-scoped CSS using the styles property

File Structure

├── app/
│ ├── app.component.ts # Root component (container)
│ └── counter.component.ts # Counter logic & UI

How It Works

src/main.ts - Bootstrap

bootstrapApplication(AppComponent, { providers: [provideAnimations()] })
  • Starts the app with the root AppComponent
  • Provides animation support via dependency injection

src/app/app.component.ts - Root Component

  • Simple container that renders <app-counter>
  • Demonstrates component composition with imports: [CounterComponent]

src/app/counter.component.ts - Counter Logic

State Management:

count = signal(0); // Reactive state primitive

Methods:

  • increment() - Increases count
  • decrement() - Decreases count (won't go below 0)
  • reset() - Sets count to 0

Template Features:

  • {{ count() }} - Displays current value (signals are called like functions)
  • (click)="decrement()" - Event binding
  • Responsive styling with gradients and hover effects

Why This Architecture?

✓ Signals are more performant than BehaviorSubject for focused reactivity

✓ Standalone Components reduce bundle size and complexity

✓ No module boilerplate - faster development

✓ Scoped styles - encapsulation prevents style conflicts

✓ Modern Bootstrap - leverages latest Angular APIs
